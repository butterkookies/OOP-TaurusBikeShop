Fixed two bugs in the customer review submission flow:

### Issue 1: Unable to submit review (FK constraint violation)
- **Root cause:** `ProductDetails.cshtml` hardcoded `OrderId = 0` when rendering the review form partial. The `FK_Review_Order` constraint rejected the insert since no Order with ID 0 exists.
- **Fix:** Added `ReviewOrderId` property to `ProductDetailViewModel`, created `GetVerifiedPurchaseOrderIdAsync()` in `ReviewRepository`, wired it through `ProductService`, and passed the real OrderId in the view.
- **Files changed:** `ReviewRepository.cs`, `ProductDetailViewModel.cs`, `ProductService.cs`, `ProductDetails.cshtml`

### Issue 2: Review text disappears when submitting without a rating
- **Root cause:** `_ReviewForm.cshtml` (partial on product detail page) had no client-side validation. Submitting without a rating caused a server redirect, losing the typed text.
- **Fix:** Added JS that prevents form submission and shows a styled warning if no star rating is selected. Warning auto-hides once a rating is picked.
- **Files changed:** `_ReviewForm.cshtml` (added inline `<style>` + `<script>`)

> Note: `Review.cshtml` (standalone write-a-review page) already had this validation â€” only the partial needed the fix.
