The Updated XAML Refactor Prompt
Role: Expert WPF UI/UX Architect.

Context: I am refactoring a View to match a strict Design System. I have provided my AppStyles.xaml which uses a Token-Based System for layout and styling.

The Task: Refactor the provided XAML file to be 100% compliant with the AppStyles.xaml architecture. Ensure the UI looks professional, balanced, and modern.

Mandatory Constraints (Do Not Deviate):

Layout Tokens: Use {StaticResource Space4}, {StaticResource Space8}, etc., for all Margin and Padding properties. Never use hardcoded numbers.

Color & Brush Tokens: * Use BgBrush for main backgrounds and SurfaceBrush for cards/containers.

Use TextPrimaryBrush (Gray100) for headers and TextSecondaryBrush (Gray300) for descriptions.

Use AccentBrush (#8AB4F8) only for primary actions or focus highlights.

Typography: Apply the specific Text Styles: TextTitle, TextSection, TextBody, or TextCaption. Do not manually set FontSize or FontWeight on TextBlocks.

Control Styles: * Buttons must use Style="{StaticResource PrimaryButton}" or SecondaryButton.

Inputs must use Style="{StaticResource TextBoxStyle}".

Containers should use Style="{StaticResource Card}" which handles the RadiusLG and SurfaceBrush automatically.

Interactive Logic: Ensure you utilize the RadiusMD (8px) for consistency across custom borders if the Card style is not applicable.

Deliverable: Provide the complete, refactored XAML code. Prioritize visual hierarchy, consistent spacing (using the Space scale), and a clean "Dark Mode" aesthetic consistent with the Gray900/800 palette.
